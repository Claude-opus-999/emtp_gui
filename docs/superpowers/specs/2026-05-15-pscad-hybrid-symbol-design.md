# PSCAD Hybrid 元件符号设计

> 日期：2026-05-15  
> 适用范围：EMTP GUI 画布元件符号第一阶段改造  
> 设计目标：让元件符号学习 PSCAD 等成熟 EMTP 软件的工程表达方式，同时保持当前模型、连线和求解链路稳定。

## 1. 背景

当前 `ui/circuit_canvas.py` 中的元件符号直接绘制在 `ComponentGraphicsItem` 内。基础元件、线路、电缆、探针、UMEC 变压器共用一套较简单的彩色线稿，已经能表达基本元件类型，但和成熟 EMTP/PSCAD 工具相比还有三个不足：

- 视觉上偏“彩色图标”，工程图纸感不足。
- 复杂元件，如 ULM、LCP、UMEC，不能充分表达相数、端口组和接法。
- 探针符号没有形成 PSCAD 式测量元件语言，电流方向、节点电压、对地电压的差异不够直观。

第一阶段采用“PSCAD Hybrid”方向：基础电气元件使用黑白工程线稿，复杂线路/电缆/变压器使用轻量工程块，探针使用 PSCAD 式蓝色测量箭头。

## 2. 设计原则

1. 主体使用黑白工程线稿。颜色只用于选中态、探针语义和必要状态提示。
2. 端口优先。端口位置必须与模型引脚一致，复杂元件显示端口组，不隐藏相数。
3. 标签小而近。元件名称、相数、`Vg` / `V` / `I` 等语义标签贴近符号，避免大面积装饰文字。
4. 可打印、可缩放。符号在默认画布缩放、缩略视图和黑白打印中仍可区分。
5. 不复制 PSCAD 图标资产。只学习其工程表达方式，绘制原创符号。

## 3. 第一阶段覆盖范围

第一阶段覆盖 5 类符号：

1. 基础二端口：`RESISTOR`、`INDUCTOR`、`CAPACITOR`、`SERIES_RL`、`SWITCH`、`MOA`、`LPM`。
2. 源类：`VOLTAGE_SOURCE`、`CURRENT_SOURCE`。
3. 探针：对地电压探针、两节点电压探针、电流探针。
4. 线路/电缆工程块：`BERGERON`、`ULM`、`LCP_OHL`、`LCP_SINGLE_CABLE`、`LCP_THREE_CABLE`。
5. 变压器/子电路块：`UMEC_TRANSFORMER`、`SUBCIRCUIT`。

本阶段不改变求解器 API、不改变项目文件格式语义、不增加新的电气模型。必要时会调整引脚生成逻辑，使画布可见端口与电气含义一致。

## 4. 符号族设计

### 4.1 基础二端口

基础二端口符号使用传统黑白电路图表达：

- 电阻：锯齿线，两端水平端口。
- 电感：线圈弧线，两端水平端口。
- 电容：双极板，两端水平端口。
- 串联 RL：左侧短锯齿 + 右侧短线圈。
- 开关：打开触点符号，保留两端端口。
- MOA：避雷器/非线性元件线稿，减少彩色填充。
- LPM：火花间隙 + 闪络提示，保持二端口。

这些符号保持当前 `nf` / `nt` 引脚语义，不改变节点映射。

### 4.2 源类

源类保留圆形源符号，但强化方向信息：

- 电压源显示 `+` / `-`，对应 `node_pos` / `node_neg`。
- 电流源显示箭头，箭头方向与符号默认端口方向一致。
- 端口圆点固定在左右两端，避免连线与符号主体重叠。

### 4.3 探针

探针学习用户提供的 PSCAD 测量符号：

- 电流探针：蓝色水平箭头，标签使用探针实例名，默认显示 `I` 或当前名称，如 `Ia_1`。
- 两节点电压探针：蓝色向上箭头，两个端口分别对应正端和负端，标签使用探针实例名，默认显示 `V`。
- 对地电压探针：蓝色向上箭头 + 接地符号，单端口对应被测节点，默认显示 `Vg`。

探针不再画成圆形按钮式图标。蓝色只表达测量语义，端口和箭头必须清楚表达测量方向。

### 4.4 线路和电缆工程块

线路和电缆使用轻量矩形工程块：

- Bergeron：双端线路块，显示线路方向和 `nk` / `nm` 端口。
- ULM：按相数显示多条平行相线，端口组与 `nk_i` / `nm_i` 对应。
- LCP-OHL：类似 ULM，但标注 `LCP`，内部可显示相线和地线组。
- LCP 单芯电缆：显示电缆组，每根电缆内部表达 core / sheath / armor。
- LCP 三芯电缆：显示三芯 + 护套/管道导体组。

这些符号的首要目标是显示“这是多导体线路模型”，而不是渲染真实截面图。真实截面仍由 LCP 参数预览窗口承担。

### 4.5 UMEC 变压器

UMEC 必须学习用户提供的 PSCAD 风格，而不是简单的普通变压器块。

UMEC 符号为矩形组件：

- 左侧为 #1 绕组，右侧为 #2 绕组。
- 左边外侧显示 `A`、`B`、`C`，对应 `H_A`、`H_B`、`H_C`。
- 右边外侧显示 `A`、`B`、`C`，对应 `X_A`、`X_B`、`X_C`。
- 框内显示 `umec`、`S_mva [MVA]`、`#1`、`#2`、`V1_kV [kV]`、`V2_kV [kV]`。
- 每侧内部绕组图由对应接法决定：
  - `Y`：画 Y 符号，并显示该侧底部中性点端口。
  - `Y_gnd`：画 Y 符号，并显示该侧底部接地点端口。
  - `Delta`：画三角形，不显示该侧底部端口。

UMEC 引脚必须动态重建：

- #1 侧始终包含 `H_A`、`H_B`、`H_C`。
- #1 侧当 `wtype1` 为 `Y` 或 `Y_gnd` 时包含 `H_N`；当 `wtype1` 为 `Delta` 时不包含 `H_N`。
- #2 侧始终包含 `X_A`、`X_B`、`X_C`。
- #2 侧当 `wtype2` 为 `Y` 或 `Y_gnd` 时包含 `X_N`；当 `wtype2` 为 `Delta` 时不包含 `X_N`。

当属性面板或 UMEC 参数对话框改变 `wtype1` / `wtype2` 时，组件引脚必须重建，并删除连接到已消失引脚的连线。`Y_gnd` 侧虽然内核映射到地，但画布上仍显示底部端点，和用户提供的 PSCAD 示例一致。

### 4.6 子电路

子电路继续使用块符号，但调整为克制的黑白工程块：

- 块内显示子电路名称。
- 端口标签靠近端口，避免遮挡连线。
- 端口方块或小圆点保持统一风格。

## 5. 架构设计

当前 `ComponentGraphicsItem` 已经承担过多绘制职责。第一阶段改造时应提取一个符号绘制模块，例如：

- `ui/symbols/__init__.py`
- `ui/symbols/style.py`
- `ui/symbols/primitive_symbols.py`
- `ui/symbols/probe_symbols.py`
- `ui/symbols/line_symbols.py`
- `ui/symbols/umec_symbols.py`

`ComponentGraphicsItem.paint()` 只负责：

1. 准备 painter、选中态和通用上下文。
2. 根据 `ComponentType` 分派到符号绘制函数。
3. 绘制通用标签和端口。

符号绘制函数只负责画符号主体，不直接改模型、不生成节点、不修改参数。

UMEC 引脚生成逻辑应集中在 `models/component_lib.py`：

- `get_pins(ComponentType.UMEC_TRANSFORMER, ..., params=...)` 或新增专用 helper。
- `create_component_pins()` 需要能够根据 `wtype1` / `wtype2` 生成动态引脚。
- 属性面板和 UMEC 参数对话框更新接法后，都调用同一个 helper 重建引脚。

## 6. 数据流和交互

### 6.1 放置元件

用户从左侧元件库放置元件时：

1. 读取默认参数。
2. 根据参数生成引脚。
3. 创建 `ComponentInstance`。
4. 画布用新的符号绘制模块渲染。

### 6.2 修改 UMEC 接法

用户修改 `wtype1` / `wtype2` 时：

1. 保存 undo 快照。
2. 更新参数。
3. 根据新接法重建 UMEC 引脚。
4. 删除连接到失效引脚的连线。
5. 通知模型刷新。

### 6.3 修改探针类型

用户修改 `probe_type` 时：

1. 保存 undo 快照。
2. 更新参数。
3. 根据 `probe_type` 重建探针引脚。
4. 删除连接到失效引脚的连线。
5. 通知模型刷新。

这同时解决当前审查中提到的“属性面板切换探针类型不重建引脚”问题。

## 7. 错误处理

- 重建引脚后，如果旧连线连接到不存在的引脚，自动删除这些连线。
- 删除连线后在状态栏或控制台输出简短提示，说明因为接法变化移除了失效连线。
- 如果旧工程文件中存在 `Delta` 接法但仍保存了 `H_N` / `X_N` 连线，加载后首次参数刷新或符号重建时清理。
- 如果 UMEC 参数缺少 `wtype1` / `wtype2`，使用默认值 `Y_gnd` / `Delta`。

## 8. 测试策略

使用测试驱动实现。

优先测试：

1. UMEC 动态引脚：
   - `Y_gnd` / `Delta` 时包含 `H_N`，不包含 `X_N`。
   - `Y` / `Y` 时包含 `H_N` 和 `X_N`。
   - `Delta` / `Delta` 时不包含 `H_N` 和 `X_N`。
2. UMEC 接法变化清理连线：
   - 从 `Y` 改为 `Delta` 后，连接到该侧中性点的连线被移除。
3. 探针动态引脚：
   - 对地电压和电流探针只有 `sense`。
   - 两节点电压探针有 `sense` 和 `ref`。
4. 符号渲染 smoke test：
   - 每类典型元件渲染到 `QImage` 后非空。
   - UMEC 三种接法渲染结果不同。
   - 三种探针渲染结果不同。
5. 全量回归：
   - `python -m unittest discover -v`。

## 9. 非目标

第一阶段不做以下事情：

- 不实现 PSCAD 全量符号库。
- 不导入或复制 PSCAD 图标资源。
- 不改变求解器计算行为。
- 不重做整体 UI 主题。
- 不实现 LCP 真实截面图在画布符号内的完整几何渲染。

## 10. 验收标准

设计实现完成后应满足：

- 基础元件、源、探针、线路/电缆、UMEC、子电路符号整体呈现 PSCAD 式工程线稿风格。
- 三种探针在画布上能一眼区分电流、两节点电压、对地电压。
- UMEC 内部绕组图和底部端点随 `wtype1` / `wtype2` 动态变化。
- UMEC 的画布可见端点与模型引脚一致。
- 所有新增和既有测试通过。
